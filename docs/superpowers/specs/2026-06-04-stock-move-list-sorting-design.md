# Inject 批量移仓列表排序 · 设计文档

> 日期: 2026-06-04 · 状态: 已确认设计, 进入 writing-plans / 实现

## 目标

为 `Inject` 页面里的 `Batch Stock Move` 分组列表增加列排序能力。

用户应当能够点击表头，对当前可见的分组行按升序或降序排序，便于快速按名称、品质、类型、CID、尺寸、数量或占格数筛选目标藏品。

## 当前状态

`src/inject/StockMovePanel.vue` 当前已经具备：

- 载入物品箱明细快照
- 选择来源/目标物品箱
- 把同 `itemCid` 的藏品分组展示
- 搜索过滤
- 可见项全选
- 顺序批量移动与实时进度

当前列表顺序是固定的：

- 先按 `boxCount` 降序
- 再按藏品名称升序
- 最后按 `itemCid` 升序兜底

这能满足默认展示，但无法支持针对某个字段快速重排。

## 已确认行为

1. 除首列勾选框外，全部 7 个数据列都支持排序：
   - 名称
   - 品质
   - 类型
   - CID
   - 尺寸
   - 数量
   - 占格
2. 点击列表头时，只维护一个激活排序列。
3. 同一列重复点击时，在 `desc` 与 `asc` 之间切换。
4. 初始默认顺序保持现有行为不变，也就是 `占格 DESC -> 名称 ASC -> CID ASC`。
5. 搜索仍先按关键字过滤，再对过滤后的结果应用当前排序。
6. 勾选状态继续按 `itemCid` 保存；切换排序后，已选项不能丢失。
7. 本次只改 renderer 层，不改 agent、preload、desktop bridge 或协议。

## 方案比较

### A. 推荐方案: 单列排序, 双态切换

在 `StockMovePanel.vue` 中增加 `sortKey` 与 `sortDirection` 状态；表头点击后切换当前排序字段与方向。

优点：

- 改动小，完全局限在现有组件
- 易于和当前搜索、分组、多选逻辑组合
- 测试边界清楚
- 不会引入新的协议或状态同步复杂度

缺点：

- 不支持用户自定义多字段组合排序

### B. 单列排序, 三态切换

同一列表头按 `默认 -> desc -> asc -> 默认` 循环。

优点：

- 用户可以直接从任意列回到“无显式排序”

缺点：

- 表头状态和测试都更复杂
- 当前列表已经有稳定默认顺序, 三态收益不大

### C. 多列排序

允许用户依次叠加多个排序字段。

优点：

- 表达能力最强

缺点：

- 明显超出当前面板复杂度
- 交互提示、状态管理和测试成本都偏高

### 结论

采用 **A**。

## 设计

### 1. 数据模型

排序状态仅保存在 `src/inject/StockMovePanel.vue` 内部：

- `sortKey`
- `sortDirection`

不新增全局 store，不写入本地存储，也不向父组件暴露。

`sourceGroups` 的职责调整为：

- 先完成分组
- 再应用统一的排序函数

`visibleSourceGroups` 继续只负责搜索过滤后的结果输出，不承担状态切换逻辑。

### 2. 排序规则

支持的排序 key：

- `name`
- `quality`
- `type`
- `itemCid`
- `size`
- `count`
- `boxCount`

比较规则：

- 文本列使用字符串比较，并保留 `name -> itemCid` 兜底，避免顺序抖动
- 数值列使用数值比较，并在相等时回退到默认比较
- `size` 不能按显示文本直接比较；主值应使用 `width * height`，相等时再回退到尺寸文本与默认比较
- 默认比较函数继续保持 `boxCount DESC -> name ASC -> itemCid ASC`

### 3. 交互

表头改成可点击按钮或等效可访问控件，要求：

- 点击列头触发排序
- 当前激活列显示方向标记
- 非激活列不显示误导性的方向状态
- 不改变现有复选框、多选、提交和搜索布局

这是一个功能增强，不做视觉重构。

### 4. 兼容性

以下行为保持不变：

- 搜索关键字匹配规则
- 同类藏品分组
- 可见项全选
- 顺序逐件搬运
- 进度统计
- 移动成功后清空选择

因为勾选值仍然是 `itemCid`，所以排序切换不会影响已选中项。

### 5. 测试

在 `src/inject/StockMovePanel.test.js` 增加行为测试，至少覆盖：

1. 默认顺序仍保持 `boxCount DESC`
2. 点击数值列可切换升序/降序
3. 点击文本列可切换升序/降序
4. 搜索过滤后, 排序继续作用于可见结果
5. 排序切换后，已勾选项保持选中

不需要新增 agent、Electron 或 IPC 层测试。

## 验收标准

1. 用户载入物品箱后，批量移仓列表的 7 个数据列都可点击排序。
2. 默认载入时，列表顺序和当前版本一致。
3. 点击 `数量`、`占格`、`CID` 这类数值列时，可在升序/降序之间切换。
4. 点击 `名称`、`品质`、`类型` 这类文本列时，可在升序/降序之间切换。
5. `尺寸` 排序按占格面积而不是纯字符串比较。
6. 搜索后再排序，只有可见结果参与重排。
7. 已勾选的行在排序前后保持选中。
8. 批量移动、搜索、全选和进度功能不回归。
